이번 업데이트는 한마디로, **오픈클로가 "대화형 봇"에서 "실제 작업을 굴리는 운영 시스템" 쪽으로 더 확실히 이동한 변화**입니다.

먼저 핵심만 보면

- **ClawHub가 기본 스킬 스토어**가 되면서 스킬 설치 흐름이 더 단순해졌고
- **에이전트 세션 유지 시간이 48시간**으로 늘어나 긴 작업이 훨씬 안정적이 되었고
- **ChatGPT 5.4가 기본 모델**로 올라가 기본 성능이 개선됐고
- **플러그인 승인 시스템**이 들어와 위험 작업 전 확인 절차가 생겼고
- **ACP 코딩 세션 지원 채널이 확대**되어 실제 작업 투입이 쉬워졌고
- **xAI(Grok) 웹 검색 통합**으로 실시간 X 데이터 활용 범위가 넓어졌고
- **보안 패치가 대거 반영**되며 운영 안정성이 강화됐습니다

즉, 이번 업데이트는 기능 몇 개가 추가된 수준이 아니라, **멀티봇 운영·자동화·안전성·지속 작업 흐름**이 전반적으로 강화된 업데이트에 가깝습니다.

---

## 1. 스킬 설치 방식: ClawHub가 기본이 됐어요

이전에는 스킬을 npm 중심으로 설치했다면, 이제는 **ClawHub가 기본 스킬 스토어** 역할을 합니다.

쉽게 말하면, 스킬 설치가 예전보다 **"패키지 설치"보다 "스토어에서 가져오기"**에 가까워진 겁니다. 처음 쓰는 사람 입장에서는 어떤 스킬이 있는지 찾고 붙이는 흐름이 더 직관적입니다.

**체감 포인트**
- 스킬 탐색이 쉬워짐
- 설치 진입장벽이 낮아짐
- 봇 기능 확장이 더 앱스토어처럼 느껴짐

## 2. 세션 타임아웃: 10분에서 48시간으로 확대

이건 실사용 기준으로 꽤 큰 변화입니다.

기존에는 잠깐 응답이 끊기면 세션이 쉽게 끝나 작업 흐름이 끊기는 경우가 있었는데, 이제 **48시간까지 유지**됩니다.

**왜 중요하냐면**
- 긴 코딩 작업
- 문서 정리 작업
- 중간에 사람이 확인하고 이어가는 작업

같은 흐름에서 끊김이 크게 줄어듭니다. 즉, 봇을 **짧게 툭 묻는 용도**뿐 아니라, **길게 맡겨두는 작업 파이프라인**으로 쓰기 쉬워졌습니다.

## 3. 기본 모델 업그레이드: ChatGPT 5.4 적용

기본 모델이 GPT-5.2에서 **ChatGPT 5.4**로 올라갔습니다.

이 변화는 겉보기에는 단순하지만, 실제로는 꽤 중요합니다. 별도 설정 없이도 더 나은 기본 성능을 기대할 수 있기 때문입니다.

**체감상 기대되는 부분**
- 지시 이해력 개선
- 복잡한 작업 정리 능력 향상
- 기본 응답 품질 안정화

즉, "설정 안 해도 기본기가 좋아진 업데이트"에 가깝습니다.

## 4. 플러그인 승인 시스템: 안전장치가 눈에 띄게 강화

이번 업데이트에서 가장 반가운 변화 중 하나는 **승인 시스템 추가**입니다.

이제 봇이 위험하거나 민감한 도구를 실행하려 하면, 먼저 사용자에게 **"정말 실행할까?"**를 확인할 수 있습니다. 텔레그램 버튼, 디스코드, `/approve` 같은 방식으로 승인 흐름을 처리할 수 있게 됐습니다.

**이게 중요한 이유**
- 자동화가 강해질수록 실수 비용도 커지는데
- 그 전에 사람이 개입할 수 있는 안전 레이어가 생겼기 때문입니다

즉, 더 강한 자동화와 더 강한 통제를 같이 가져가려는 방향으로 볼 수 있습니다.

## 5. ACP 코딩 세션 확장: 더 많은 채널에서 바로 작업 가능

ACP 코딩 세션이 디스코드, iMessage, BlueBubbles 등 더 많은 채널로 확장됐습니다.

이 말은 곧, 굳이 별도 환경으로 옮기지 않아도 **현재 대화가 오가는 맥락 안에서 바로 코딩 작업을 붙일 수 있다**는 뜻입니다.

**체감 변화**
- 작업 진입 속도 빨라짐
- 대화와 실행의 거리 감소
- 협업 채널 안에서 바로 작업 연결 가능

오픈클로가 단순 응답 도구가 아니라 **채널 안에서 실제로 일하는 에이전트**에 가까워진 변화라고 볼 수 있습니다.

## 6. xAI(Grok) 웹 검색 통합: X 데이터 활용 강화

Grok의 `x_search`가 기본 지원되면서, 설정만 하면 **X(트위터) 기반 실시간 검색**을 붙일 수 있게 됐습니다.

이건 특히 아래 같은 용도에서 의미가 큽니다.

- 실시간 반응 확인
- 커뮤니티 트렌드 탐색
- 제품 출시 직후 여론 조사
- SNS 중심 이슈 모니터링

즉, 일반 웹 검색만으로는 늦거나 약한 부분을 **실시간 소셜 흐름**으로 보완할 수 있게 된 셈입니다.

## 7. 보안 패치 30개+: 운영 안정성 보강

이번 릴리스에는 **30개 이상의 보안 패치**도 함께 들어갔습니다.

여기에는
- 웹 검색 키 감사 강화
- RPC 메타데이터 보안 강화
- 무한 에코 루프 방지

같은 운영 안정성 관련 수정이 포함됩니다. 이런 변화는 눈에 잘 안 띄지만, 실제로 여러 봇을 묶어 굴리거나 채널 자동화를 돌릴수록 더 중요합니다.

한마디로, **겉보다 속이 강해진 업데이트**입니다.

---

## 주의할 점: Breaking Change도 있습니다

좋은 점만 있는 건 아닙니다. 이번에 **12개 Breaking Change**가 포함됐습니다.

대표적으로는:
- `CLAWDBOT_*`, `MOLTBOT_*` 같은 구형 환경변수 제거
- 레거시 Chrome 확장 릴레이 경로 삭제
- Qwen 포털 OAuth 제거

즉, 예전 방식으로 오래 굴리던 환경이라면 업데이트 후 일부 설정이 깨질 수 있습니다.

### 업데이트 후 권장 절차

```bash
npm i -g openclaw@latest
openclaw doctor --fix
openclaw gateway restart
```

특히 `openclaw doctor --fix`는 꼭 한 번 돌리는 게 좋습니다.

## 초반 버그는 이미 수정됐어요

v2026.3.22 직후에는 몇 가지 문제가 있었지만, 현재는 후속 버전에서 이미 해결됐습니다.

- **Dashboard 503 에러** → v2026.3.23에서 수정
- **WhatsApp 연결 끊김** → v2026.3.23-2에서 수정

지금 기준으로는 **최신 v2026.3.28 이상으로 올리면 안정적으로 쓰는 쪽**에 가깝습니다.

## 결론

이번 3월 업데이트는 "기능이 좀 늘었다" 정도로 보기엔 아쉽습니다.

핵심은 오픈클로가 **더 오래 일하고, 더 안전하게 승인받고, 더 많은 채널에서 실제 작업을 수행하고, 그 결과를 운영 흐름으로 연결하기 쉬워졌다**는 데 있습니다.

즉, 오픈클로는 점점 **그냥 챗봇이 아니라, 실제 업무를 굴리는 자동화 운영 플랫폼** 쪽으로 진화하고 있습니다.
